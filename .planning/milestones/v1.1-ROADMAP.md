## Milestone Summary

**Decisões Tomadas:**

| Decisão | Racional |
|---------|---------|
| Import playwright com try/except no módulo | Permite mock via `patch()` nos testes — design para testabilidade |
| `browser.close()` em `finally` | Garante fechamento de recursos mesmo em exceções |
| `domcontentloaded` (não `networkidle`) | Mais confiável em portais com tracking scripts |
| `empresa_nova: bool` como campo de primeiro nível | Mantém compatibilidade com `supplier_analyzer.py` que lê `flags_risco` |
| `return resultado` (não `return merged`) | Preserva semântica de retorno — callers esperam CNPJs processados, não acumulado total |
| `total_itens` como alias de `total_empenhos` | Evita duplicação de `len(alertas_df)` |
| Campos BPS na raiz do summary dict (não dentro de `totais`) | Separação de concerns — métricas de qualidade vs. contagens de alerta |
| `alertas_por_tier` via `value_counts()` | Inclui todos os tiers automaticamente, inclusive `SEM_REFERÊNCIA` |

**Issues Resolved:**

- STAB-01: `--threshold` CLI forwarded para AIAnalyzer (era hardcoded 30)
- STAB-02: BPS URL dinâmica funciona via Playwright headless
- STAB-03: `empresa_nova` calculado com dados BrasilAPI reais (edge cases cobertos)
- STAB-04: `numpy` em requirements.txt (era dependência transitiva silenciosa)
- STAB-05: `--ano all` preserva fornecedores existentes via merge por CNPJ
- STAB-06: VALIDATION.md retroativo para Phases 0, 1, 3

- Cache de disco com CNPJ real (`46634044000174`) causava falha silenciosa nos testes → resolvido com mocks de `carregar_cache()`/`salvar_cache()`
